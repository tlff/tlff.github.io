//创建软连接
ln -s /usr/local/php73/bin/php /usr/bin/php
ln -s /usr/local/php73/sbin/php-fpm /usr/bin/php-fpm

# 让nginx使用新的php

因为PHP使用php-fpm管理的nginx只是监听127.0.0.1:9000这个端口,所以有两个方法可以让nginx使用新的PHP

  1. 关了旧的php-fpm,使用新的php-fpm
  2. 修改nginx监听的端口同时修改fpm配置中的端口,启动新的php-fpm

# 配置php-fpm

编译安装的php-fpm在/usr/local/php73/sbin,默认的配置文件在/usr/local/php73/etc中

从默认配置文件创建一个配置文件

cp /usr/local/php73/etc/php-fpm.conf.default /usr/local/php73/etc/php-fpm.conf
cp /usr/local/php73/etc/php-fpm.d/www.conf.default /usr/local/php73/etc/php-fpm.d/www.conf

修改www.conf中的user和group 为nginx

# 安装php途中遇到的问题

  • configure: error: off_t undefined; check your library configuration

    解决:将libzip加入的ld path中 并在设置参数的时候指定libzip的安装目录

  • 遇到某些扩展没有 一般安装这个扩展和 扩展-devel版本

# 编译安装libzip

yum中自带的libzip版本太低只能手动装了

wget https://nih.at/libzip/libzip-1.2.0.tar.gz
tar -zxvf libzip-1.2.0.tar.gz
cd libzip-1.2.0
./configure --prefix=/usr/local/libzip
make && make install
//将libzip加入到系统的ld path中 后面这句是刷新缓存
echo '/usr/local/libzip/lib' >> /etc/ld.so.conf && ldconfig
